## 📖 项目简介

本作品基于沁恒 **CH32V307** 核心板，设计了一款面向智能拍摄与辅助科技场景的**多模态交互三自由度稳像云台系统**。

系统通过摄像头实时采集图像，采用轻量化卷积神经网络实现**手势识别**与**人脸检测**，触发三自由度云台控制。创新性地集成**语音指令控制模块**与**低功耗蓝牙通信模块**，构建手势+语音+视觉的多模态人机交互体系，并采用**增量式PID算法**实现三轴舵机的精准控制。硬件上集成高扭矩舵机、OLED状态反馈模块、语音模块、低功耗蓝牙通信模块及独立电源，依托CH32V307的外设资源，通过多组UART串口实现识别指令与舵机控制的高效交互。创新点在于多模态指令映射、三轴协同控制、语音-视觉跨模态控制技术及低功耗边缘计算架构，适用于需要非接触控制的智能拍摄场景，以及益于特殊人群的辅助科技应用，兼具高实用性与可扩展性。

## 💡 项目背景与设计动机

### 为什么做这个项目？

在当今数字化时代，**摄像设备的智能化控制**已成为一个重要的技术方向。传统的云台控制方式——无论是手持遥控器还是手机APP——都要求操作者与设备保持物理接触或近距离操作。然而，在许多实际场景中，这种传统的交互方式存在明显的局限性：

1. **影视拍摄场景**：摄影师在进行多机位拍摄时，双手往往被其他设备占用，无法同时操控云台；在需要特殊角度拍摄时，操作者可能需要远离设备以获得更好的观察视角。

2. **无障碍辅助场景**：对于肢体残障人士而言，传统的按键式或触屏式交互方式可能无法使用。他们迫切需要一种**非接触式、直觉化**的人机交互方式来操控辅助设备。

3. **工业巡检场景**：在高温、高辐射、有毒等危险环境中，操作人员需要在安全距离外远程操控检测设备，传统的有线控制方式存在线缆束缚和信号延迟的问题。

4. **教育科研场景**：嵌入式系统、计算机视觉、控制理论等课程需要一个综合性的实践平台，让学生能够将理论知识应用于实际系统。

### 设计理念

基于上述需求分析，我们确立了以下设计原则：

- **多模态融合**：不依赖单一交互方式，而是同时支持手势、语音、视觉、蓝牙四种控制模式，用户可以根据场景灵活选择最合适的方式。
- **边缘计算优先**：将计算密集型任务（如图像识别）交给PC端处理，实时控制任务（如PID运算、舵机驱动）由嵌入式端完成，实现"边缘-云端"协同计算。
- **模块化架构**：硬件和软件均采用模块化设计，便于后期功能扩展和维护升级。
- **实时性保证**：关键控制路径采用中断驱动，确保从传感器输入到执行器输出的延迟最小化。

## 🎯 功能特性

### 🤚 多模态手势识别

手势识别是本系统最核心的交互方式之一。我们采用 Google 的 **MediaPipe** 框架作为手势识别引擎，该框架基于轻量化的卷积神经网络，能够在普通PC上实现实时的21点手部关键点检测。

**识别原理详解：**

MediaPipe 首先从视频帧中检测手部区域，然后定位21个手部关键点（包括指尖、指关节、手掌中心等）。通过分析这些关键点的空间位置关系，我们可以判断每根手指是否伸出：

- 对于拇指：比较指尖（关键点4）与拇指根部（关键点2）的水平位置关系
- 对于其他四指：比较指尖与对应掌指关节的垂直位置关系
- 如果指尖在手掌凸包外部，则认为该手指处于"伸出"状态

通过组合不同手指的伸出状态，系统可以识别出多种手势。为了提高识别的鲁棒性，我们还加入了**稳定性检测**机制：连续3帧识别到相同手势才确认为有效手势，避免因手部抖动导致的误识别。

支持 **7种手势** 识别，每种手势映射不同控制指令：

**手势-功能映射表：**

| 手势 | 功能 | 云台动作 | 识别条件 |
|:---:|:---:|:---:|:---:|
| ☝️ 数字 `1` | X轴正转 | 水平向右 +5° | 仅食指伸出，关节角度>160° |
| ✌️ 数字 `2` | X轴反转 | 水平向左 -5° | 食指+中指伸出 |
| 🤟 数字 `3` | Y轴正转 | 垂直向上 +5° | 拇指+食指+中指伸出，指尖距离>60px |
| 🖐️ 数字 `4` | Y轴反转 | 垂直向下 -5° | 四指伸出（除小指） |
| 🖐️ 数字 `5` | 启动追踪 | 开启人脸跟踪模式 | 五指全部伸出 |
| 🤙 数字 `6` | Z轴正转 | 航向顺时针 +5° | 拇指+小指伸出（"打电话"手势） |
| 🤏 数字 `7` | Z轴反转 | 航向逆时针 -5° | 拇指+食指+中指伸出，指尖距离<60px |
| ✊ 握拳 `fist` | 停止 | 停止所有运动 | 无手指伸出 |

**防抖动与可靠性设计：**

由于手势识别存在一定的不确定性，我们在通信层面加入了多重保障机制：

1. **稳定性窗口**：连续3帧相同手势才触发指令发送
2. **冷却时间**：手势切换后需等待500ms才能触发下一次切换
3. **重复发送**：每个手势指令重复发送4次，确保嵌入式端能够接收到
4. **变化阈值**：人脸坐标变化超过5°才发送，减少通信负载

### 👁️ 智能人脸追踪

人脸追踪是本系统的另一核心功能。当用户做出"五指张开"手势或通过语音指令激活追踪模式后，系统会自动锁定摄像头画面中的人脸，并驱动云台跟随人脸运动，使人脸始终保持在画面中心区域。

**技术实现细节：**

1. **人脸检测**：采用 OpenCV 内置的 **Haar级联分类器**（`haarcascade_frontalface_default.xml`），这是一种经典的基于特征的机器学习检测方法。虽然近年来深度学习方法在准确率上有所超越，但Haar级联在嵌入式场景下具有**速度快、资源占用少**的优势，非常适合实时应用。

2. **目标选择策略**：当画面中存在多张人脸时，系统选择**面积最大的人脸**作为跟踪目标，这通常是距离摄像头最近、也是最可能需要被拍摄的主体。

3. **坐标映射算法**：将人脸中心的像素坐标转换为云台舵机的目标角度。具体来说：
   - 图像中心（320, 240）对应云台的中位角度（94°）
   - 人脸偏离图像中心越远，舵机需要偏转的角度越大
   - 映射公式：`angle = 94 - (center - image_center) * (86 / image_center)`

4. **平滑滤波**：为了消除人脸检测结果的帧间抖动，我们采用**5点移动平均滤波器**。具体做法是维护一个长度为5的滑动窗口，将最近5帧的人脸坐标取平均值作为最终输出。这样即使某一帧的检测结果存在偏差，也不会导致云台的突然跳动。

5. **三轴协同补偿**：在跟踪人脸时，不仅X轴和Y轴需要跟随运动，Z轴（航向）也需要进行协同补偿。当人脸偏向画面左侧时，Z轴需要顺时针旋转以获得更好的拍摄角度；反之亦然。补偿公式为：`z_compensation = norm_x × 20 + norm_y × 5`。

### 🔊 语音控制系统

语音控制模块为系统提供了第三种交互维度。通过集成**LD3320**语音识别芯片，用户可以在不使用双手的情况下，通过简单的语音指令控制云台的行为。

**工作原理：**

LD3320是一款专用的语音识别芯片，内置了非特定人语音识别引擎（即不需要对特定用户进行训练）。芯片通过UART串口与CH32V307通信，当识别到预设的语音关键词后，会通过串口发送对应的识别码。

**系统中的语音控制流程：**

1. 用户说出语音指令（如"开始追踪"）
2. LD3320芯片识别语音，通过UART5发送识别结果给CH32V307
3. CH32V307接收到识别结果后，通过UART4转发给Python上位机
4. Python上位机收到指令后，启动或关闭人脸追踪模式

这种**语音→嵌入式→PC**的跨层级控制链路，实现了语音-视觉的跨模态融合控制。


### 📱 蓝牙远程控制

通过集成**CH9141**低功耗蓝牙模块，系统支持手机APP远程控制。用户可以通过蓝牙连接，实时查看传感器数据、调整控制参数、切换工作模式。

**蓝牙通信协议：**

蓝牙模块通过UART6与CH32V307通信，采用简单的命令-响应协议：

| 命令码 | 功能描述 |
|:---:|:---|
| `1` | 查看CO气体浓度（MQ7传感器） |
| `2` | 查看天然气浓度（MQ4传感器） |
| `3` | 查看测距数据 |
| `5` | 查看温度数据 |
| `6` | 查看湿度数据 |
| `7` | 当前模式阈值 -1 |
| `8` | 当前模式阈值 +1 |
| `9` | 全部传感器数据上报 |

蓝牙模块每500ms通过TIM10定时中断自动读取一次数据，确保控制指令的及时响应。

## 🛠️ 硬件配置

### 核心芯片选型

本系统选用**CH32V307VCT6**作为主控芯片，这是沁微科技推出的一款基于**RISC-V**架构的32位通用微控制器。选择该芯片主要基于以下考量：

1. **高性能**：主频高达144MHz，能够满足实时控制的计算需求
2. **丰富的外设**：集成了多路UART、SPI、I2C、ADC、PWM等外设，无需额外扩展芯片
3. **充足的存储**：256KB Flash + 64KB SRAM，为复杂的控制算法提供了足够的运行空间
4. **开源生态**：RISC-V架构具有开放的指令集和丰富的社区资源，便于开发和调试
5. **成本优势**：相比同级别的ARM芯片，CH32V307具有更好的性价比

## 🧮 核心算法详解

### 1️⃣ PID控制算法

PID（比例-积分-微分）控制是工业控制中最经典的算法之一。本系统为X/Y/Z三个轴分别配置了独立的PID控制器，每个轴的参数根据其运动特性进行了专门调校。

**算法公式：**

```
    误差(error) = 目标角度 - 当前角度
    
    P (比例项) = Kp × error
        → 对误差进行比例响应，误差越大响应越强
    
    I (积分项) = Ki × Σerror (带积分限幅)
        → 消除稳态误差，当|error|<10°时才累积积分
        → 当|error|≥10°时清零积分，防止积分饱和
    
    D (微分项) = Kd × (error - prev_error)
        → 预测误差变化趋势，抑制超调和振荡
    
    输出 = P + I + D (带输出限幅)
```

**三轴PID参数配置：**

| 轴 | Kp | Ki | Kd | 积分限幅 | 输出限幅 | 设计意图 |
|:---:|:---:|:---:|:---:|:---:|:---:|:---|
| X轴 | 1.2 | 0.04 | 0.1 | 30.0 | 10.0 | 快速响应，适配水平旋转的低惯量特性 |
| Y轴 | 0.8 | 0.03 | 0.2 | 40.0 | 8.0 | 平稳移动，避免垂直方向的重力影响导致抖动 |
| Z轴 | 0.8 | 0.05 | 0.2 | 20.0 | 5.0 | 精确控制，航向微调需要更高的精度 |

### 2️⃣ 分段比例控制（实际使用）

在实际调试中，我们发现对于本系统的舵机响应特性，**分段比例控制**比完整的PID控制更加适合。这是因为：

1. 舵机本身具有位置保持能力，不需要积分项来消除稳态误差
2. 舵机的响应速度有限，微分项的预测作用不明显
3. 分段控制可以根据误差大小自动切换响应速度，更加符合人机交互的直觉

**分段策略：**

```
    ┌─────────────────────────────────────────────────────┐
    │               分段比例控制策略                        │
    ├─────────────────────────────────────────────────────┤
    │                                                     │
    │   |误差| > 20°  ──▶  step = error / 2   快速追赶   │
    │                     大误差时全速运动，快速接近目标    │
    │                                                     │
    │   |误差| > 5°   ──▶  step = error / 3   中速移动   │
    │                     中等误差时适当减速，避免过冲     │
    │                                                     │
    │   |误差| ≤ 5°   ──▶  step = ±1         精细微调   │
    │                     小误差时逐度调整，精确定位       │
    │                                                     │
    └─────────────────────────────────────────────────────┘
```

这种分段策略的优点是：**大误差时快速响应，小误差时精细控制**，既保证了系统的动态性能，又提高了定位精度。

### 3️⃣ 三轴协同控制算法

在人脸跟踪模式下，三个轴不是独立运动的，而是需要协同配合。特别是Z轴（航向），需要根据X/Y轴的偏移量进行补偿，以获得最佳的拍摄角度。

**协同补偿公式：**

```
    X轴偏移 ──┐
              │
              ├──▶ Z轴补偿 = norm_x × 20 + norm_y × 5
    Y轴偏移 ──┘

    其中：
    - norm_x = (face_x - 135) / 135.0  (归一化到[-1, 1])
    - norm_y = (face_y - 120) / 120.0  (归一化到[-1, 1])
    - Z轴初始角度 = 40°
    - 补偿系数 20 和 5 根据实际测试调整
```

### 4️⃣ 语音-视觉跨模态控制

本系统的一个创新点是实现了**语音-视觉跨模态控制**。用户可以通过语音指令激活视觉追踪模式，然后系统自动切换到人脸跟踪控制，实现"语音触发、视觉执行"的无缝切换。

```
    语音指令 ──▶ 激活视觉追踪 ──▶ 人脸检测 ──▶ 云台跟随
       │
       └──▶ "开始追踪" ──▶ Python启动追踪模式
           "停止追踪" ──▶ 关闭追踪模式
```

### 5️⃣ 低功耗边缘计算架构

本系统采用"边缘-云端"协同计算架构，将不同计算需求的任务分配到最合适的计算节点：

```
    ┌─────────────────────────────────────────┐
    │           边缘计算节点 (CH32V307)         │
    │                                         │
    │   144MHz RISC-V处理器                    │
    │   ├── PID控制 (实时, <1ms)               │
    │   ├── 传感器数据采集 (实时)               │
    │   ├── 串口数据路由 (实时)                 │
    │   └── PWM舵机驱动 (50Hz)                 │
    │                                         │
    │           云端处理节点 (PC)               │
    │                                         │
    │   Python + GPU加速                       │
    │   ├── MediaPipe手势识别 (~30ms/帧)       │
    │   ├── OpenCV人脸检测 (~20ms/帧)          │
    │   └── 图像预处理与坐标计算                │
    │                                         │
    └─────────────────────────────────────────┘
```

这种架构的优势在于：
- **实时性**：嵌入式端专注于实时控制，不受图像处理延迟的影响
- **灵活性**：PC端可以随时更新识别算法，无需重新烧录固件
- **可扩展性**：未来可以将图像识别算法移植到嵌入式端，实现完全独立运行

## 📱 应用场景

### 🎬 智能拍摄

在影视制作、直播、Vlog拍摄等场景中，操作者可以通过手势或语音控制摄像头的拍摄角度，无需接触任何控制设备。特别是当摄影师需要同时操控多个机位时，多模态控制的优势尤为突出。

**典型使用流程：**
1. 摄影师做出"五指张开"手势，激活人脸追踪模式
2. 摄像头自动锁定目标人物，云台跟随其运动
3. 需要调整拍摄角度时，通过语音指令"左转"、"右转"进行微调
4. 拍摄结束后，做出"握拳"手势，停止所有运动

### 🏥 辅助科技

对于肢体残障人士，传统的交互方式可能无法使用。本系统提供的非接触式控制方式，为他们打开了一扇新的窗户。

**无障碍应用场景：**
- 手指轻微活动即可控制云台方向
- 语音指令替代复杂的按键操作
- 人脸追踪实现"看哪拍哪"的直觉式控制

### 🎓 教学实验

本系统涵盖了嵌入式系统、计算机视觉、控制理论、通信协议等多个领域的知识点，非常适合作为综合性课程设计或毕业设计的实践平台。

**可学习的知识点：**
- RISC-V架构与嵌入式C编程
- PID控制算法原理与调参
- 串口通信协议设计
- 计算机视觉基础（手势识别、人脸检测）
- 多传感器数据融合

### 4. 操作说明

1. 系统启动后，摄像头窗口自动打开
2. 在摄像头前做出手势，观察云台响应
3. 做出"五指张开"手势激活人脸追踪
4. 做出"握拳"手势停止所有运动
5. 按'q'键退出程序
